Booking return flow (updated per your feedback)
  1. User taps "Book appointment" → expo-web-browser opens shop site
  2. User books on shop's site, closes browser
  3. App returns to foreground — detects browser closed
  4. Modal appears: "Did you schedule an appointment?" [Yes / No]
  5. Yes → prompt for service type, date, time → saves appointment → pauses reminders
  6. No → dismiss modal → daily reminders continue
Key constraints & decisions
Background location
Requires user to grant "Always" permission. iOS will ask twice — once for "While Using", once to upgrade to "Always". App should explain the value clearly during onboarding.
Dev build required
Background tasks and push notifications don't work in Expo Go. A dev build (via EAS Build or local Xcode) is needed to test the full app on device.
Single vehicle UI
Multi-vehicle-ready data model (vehicles table with user_id FK), but UI exposes only one vehicle per account for now.
