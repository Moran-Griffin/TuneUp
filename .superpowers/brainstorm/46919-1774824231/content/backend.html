Backend preference?

TuneUp needs user accounts, vehicle data, and push notifications — the backend drives all of this

A

Supabase

Postgres database + auth + storage, all hosted. Great TypeScript support, generous free tier, open-source. My recommendation for this project.

Pros

  • Real SQL, easy to query
  • Built-in auth & row-level security
  • Works great with React Native

Cons

  • Less mobile-specific than Firebase
B

Firebase

Google's BaaS — Firestore NoSQL database, auth, and cloud messaging (push notifications built-in).

Pros

  • Push notifications (FCM) built-in
  • Strong React Native ecosystem
  • Familiar to many devs

Cons

  • NoSQL can get messy for relational data
  • Google can change pricing/terms
C

Custom Node/Express API

Build your own backend — full control, but significantly more work.

Pros

  • Total control over logic
  • Easier to customize notification engine

Cons

  • Auth, DB hosting, push infra all DIY
  • Much more to build and maintain
