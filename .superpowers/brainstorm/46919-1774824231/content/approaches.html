Three approaches to building TuneUp

The main decision is the React Native setup — everything else (Supabase, Expo Notifications, location) stays the same

A

Expo Managed Workflow ⭐ Recommended

Full Expo setup — no Xcode config needed until App Store submission. Geofencing via expo-location background tasks, push notifications via Expo Push Service (wraps Apple APNs).

Pros

  • Fastest to get running on your phone
  • OTA updates (no App Store re-review for JS changes)
  • Location + notifications both well-supported
  • Great TypeScript support

Cons

  • Background geofencing requires a dev build (not Expo Go)
  • Some advanced native APIs require ejecting
B

Expo Bare Workflow

Start with Expo but with full access to native iOS code from day one. More control, but requires Xcode setup upfront.

Pros

  • No ejection surprise later
  • Can use any native module immediately

Cons

  • More initial setup than managed
  • Loses some Expo convenience tooling
  • Overkill for this project's requirements
C

React Native CLI (no Expo)

Raw React Native — full native control, but you configure everything yourself: push notifications, location permissions, background tasks.

Pros

  • No Expo dependency
  • Maximum control

Cons

  • Significantly more boilerplate
  • Xcode config required immediately
  • Slower to get to a working app
