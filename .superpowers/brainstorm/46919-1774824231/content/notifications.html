Notification engine

Two distinct notification systems — location-triggered and schedule-triggered

1. Location-based detection
How it works
  1. expo-location background task monitors GPS
  2. App detects user enters a geofence around known auto shop / dealership categories (using Apple Maps POI categories)
  3. A timer starts — if user stays 5+ minutes, a local notification fires
  4. "You've been at [place name] for a bit — getting work done?"
  5. Tapping opens the quick-add log entry screen
  6. If user ignores or leaves, timer resets — no spam
Place detection strategy
Uses iOS region monitoring (circular geofences) via expo-location. Place category matching done via Apple Maps / MapKit — no third-party API needed.

Requires "Always" location permission from user.
2. Maintenance reminders (persistent until resolved)
Trigger logic
  • Oil change: due when mileage within 500mi of interval, or date ≥ 6 months since last
  • Inspection: due 30 days before anniversary of last inspection
  • Notification fires once per day if overdue
How it's silenced
  • User logs the completed work → resets vehicle record → no more notification
  • User schedules an appointment → notification pauses until appointment date passes
  • No other way to dismiss (by design)
Implementation
Supabase Edge Function runs on a daily cron. Checks all vehicles, schedules local push notifications via Expo Push Service → APNs. No always-on background process needed.
