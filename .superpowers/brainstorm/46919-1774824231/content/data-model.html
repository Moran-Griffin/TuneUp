Data model

How TuneUp stores everything in Supabase (Postgres). Does this look right?

users — managed by Supabase Auth
id (uuid, PK)
email
created_at
vehicles — one per user for now, multi-vehicle-ready
id (uuid, PK)
user_id (FK → users)
make, model, year
current_mileage
last_oil_change_date
last_oil_change_mileage
oil_change_interval_miles (default 5000)
last_inspection_date
inspection_interval_months (default 12)
created_at
maintenance_logs — full service history
id (uuid, PK)
vehicle_id (FK → vehicles)
user_id (FK → users)
type (required — oil_change | inspection | tire_rotation | brake_service | battery | other)
date (required)
mileage (optional)
shop_name (optional)
cost (optional)
notes (optional)
created_at, updated_at
appointments — scheduled future work
id (uuid, PK)
vehicle_id (FK → vehicles)
user_id (FK → users)
shop_name
shop_booking_url (deep-link target)
scheduled_date
service_type (oil_change | inspection | etc.)
status (scheduled | completed | cancelled)
created_at
Key logic: When a maintenance_log entry of type oil_change or inspection is saved, it automatically updates the corresponding fields on the vehicles row — resetting the reminder clock and silencing the notification.
